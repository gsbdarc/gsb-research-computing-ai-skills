## üñäÔ∏è Why the Terminal

Before you type a single command, understand why the command line exists ‚Äî and why it matters specifically for research.

**Research projects have structure.** Scripts, raw data, and processed outputs should never live in the same folder. A project that mixes all three is a project that cannot be reproduced ‚Äî by you, your PI, or anyone who inherits it.

```
my_project/
‚îú‚îÄ‚îÄ data/          ‚Üê raw data, never modified
‚îú‚îÄ‚îÄ scripts/       ‚Üê your code
‚îú‚îÄ‚îÄ results/       ‚Üê outputs, generated by scripts
‚îî‚îÄ‚îÄ README.md      ‚Üê what this project is and how to run it
```

**Data arrives as a dump.** A collaborator sends you a folder with 10,000 files named `scan_0042.csv`, `FINAL_v3.xlsx`, and `untitled.txt`. Organizing that by clicking takes days. The terminal handles it in seconds.

| | GUI (Finder / Explorer) | CLI (Terminal) |
|---|---|---|
| **Speed** | Click ‚Üí drag ‚Üí drop, one file at a time | `mv *_fire_*.spell fire/` ‚Äî 300 files in one second |
| **Navigate** | Browse folders by clicking | `cd`, `ls`, `pwd` ‚Äî move anywhere instantly |
| **Organize** | Rename files one by one | Wildcards and loops sort an entire dataset in seconds |
| **Power** | Limited to what the app exposes | Run scripts, schedule jobs, automate anything |

After SSH (later this morning), every command you practice right now will work identically on the Yens cluster:

| | Your laptop | Stanford Yens |
|---|---|---|
| **Cores** | ~8‚Äì16 | 32‚Äì512 per node |
| **RAM** | ~8‚Äì32 GB | 250 GB ‚Äì 3 TB per node |
| **Commands** | `$ ls` `$ cd` `$ pwd` | `$ ls` `$ cd` `$ pwd` |
| **Access** | direct | `ssh SUNetID@yen.stanford.edu` |

Same commands. More power.

**Open your terminal now:**

You should see a blinking cursor waiting for a command. That cursor is the entrance to the dungeon.

## üó°Ô∏è Main Quest

Every hero needs to know where they stand. Your terminal is open ‚Äî now claim your footing.

{: .important }
> **Quest:** Use the command line to navigate your file system and reorganize a directory of files ‚Äî without touching a mouse.

**Investigation**

Find your current working directory and list the files in it.

```bash
pwd          # print working directory ‚Äî where you are right now
ls           # list files and directories here
```

Now look for hidden files ‚Äî files whose names start with a dot are invisible to a plain `ls`:

```bash
ls -lah      # -l long format ¬∑ -a all files (including hidden) ¬∑ -h human-readable sizes
```

A **flag** modifies the behavior of a command. `-lah` is three flags combined into one: it tells `ls` to show everything, in detail, with file sizes you can actually read. You can look up any command's flags by searching online ‚Äî for example, "ls command flags".

{: .tip }
> **Tab completion:** Start typing a filename or path and press `Tab` ‚Äî the shell completes it. Press `Tab` twice to see all matches. Use it constantly; it prevents typos and saves time.

---

**Movement**

Use `cd` to change directories. `~` is your home directory; `..` means one level up.

```bash
cd ~/Desktop    # navigate to your Desktop ‚Äî once you type D, press Tab to autocomplete
                # press Tab twice to see all options if nothing completes
pwd             # confirm where you landed
cd ..           # go up one level
pwd             # confirm you moved up
```

---

**Creation**

Create a new folder on your Desktop, conjure a scroll inside it, and duplicate it.

```bash
mkdir ~/Desktop/spell_chamber        # create a new folder on your Desktop
cd ~/Desktop/spell_chamber           # enter it
touch scroll.txt                     # create a new file
cp scroll.txt scroll_copy.txt        # create a copy of that file
ls                                   # confirm both scrolls are here
```

---

**Deletion**

Seal the chamber ‚Äî remove everything you just created.

{: .warning }
> ‚öÝÔ∏è `rm` is permanent. There is no undo, no trash can. Double-check what you are removing before you run it.

```bash
rm scroll.txt                        # remove the file
rm scroll_copy.txt                   # remove the copy
rm -r ~/Desktop/spell_chamber        # remove the folder and all its contents
```
