# RF Coding Bootcamp 2026 — DARC Dungeon

Four-day hands-on course for Stanford GSB pre-doctoral researchers. Topics: CLI, SSH, the Yens cluster, SLURM, GPU jobs, LLM APIs, and AI-assisted research workflows — taught through a magic-themed dungeon game.

**Course site:** https://gsbdarc.github.io/rf-bootcamp-2026/

---

## Prerequisites

- A **GitHub account** (free at [github.com](https://github.com))
- No other software needed before Day 1 — everything else gets installed during class

---

## Student Setup

### 1. Fork this repository

Click **Fork** in the top-right corner of this page. GitHub will create a copy of the repo at `https://github.com/YOUR-USERNAME/rf-bootcamp-2026`.

### 2. Enable GitHub Pages on your fork

In your fork:

1. Go to **Settings → Pages**
2. Under **Source**, select **GitHub Actions**
3. Click **Save**

Your personal dungeon site will be live at:

```
https://YOUR-USERNAME.github.io/rf-bootcamp-2026/
```

It takes about 2 minutes for the first build. After that, every push to `main` rebuilds automatically.

### 3. Clone your fork to the Yens

```bash
git clone https://github.com/YOUR-USERNAME/rf-bootcamp-2026.git
cd rf-bootcamp-2026
```

You'll work from this directory for all Boss Gate submissions.

## How the Course Works

Each day is a **floor** of the dungeon. Every floor has **rooms** — hands-on exercises that build on each other. Complete the main quest in each room, then move on.

**Finished early?** Look for **Chests** — optional deeper challenges that unlock named **Weapons** (skills). Weapons compound across days.

**Floor progression:** Each floor ends with a **Boss Gate**. Push your Boss Gate commit to unlock the next floor on your personal site.

---

## Boss Gates

When you complete a Boss Gate, commit and push the required files to your fork's `main` branch. A grader runs automatically (GitHub Actions) and checks your submission:

| Boss Gate | What to commit | Unlocks |
|-----------|---------------|---------|
| **Gate 1** | `signature_spell.txt` | Floor 2 — The Alchemist's Lab |
| **Gate 2** | `results/mood_ring.json` | Floor 3 — The SLURM Mines |
| **Gate 3** | `results/great_scroll_sweep.csv` + `results/failed_tasks.txt` + `README.md` | Floor 4 — The GPU Fortress |
| **Gate 4** | `results/comparison.csv` + `results/privacy_ruling.md` + updated `README.md` | Graduation |

The grader result appears in your fork's **Actions** tab (green = passed, red = failed with a message). When a floor unlocks, your site rebuilds and the next floor appears.
